◎ How Orientations (Oris) Work

An ori is a saved device orientation. Register a name, record its orientation by moving the device through the poses you want it to recognize, then stop. The device continuously calculates which saved ori it's currently closest to.

Hold Button A on the device to record; release to save. Short tap = instant single sample.

Use the Gate system on messages (enable in Advanced Tools) to conditionally send based on ori state or any sensor value.

OnlyMessage sends only when the gate condition is active
NotMessage sends only when the gate condition is NOT active
ToggleSends the high value when active, low value when not
RisingScene gate — fires once when sensor crosses above the trigger threshold (non-ori only). "Trigger" = threshold, "Delta" = minimum delta.
FallingScene gate — fires once when sensor crosses below the trigger threshold (non-ori only). "Trigger" = threshold, "Delta" = minimum delta.

Calibrate & Zero
🎯
Orientation Zero
Tell the device what direction is "home." Do this whenever you put the sensor on, move it to a new position, or the orientation output looks wrong.
Zero Now
Takes a single snapshot of the current orientation and sets it as zero. Works instantly — no need to hold still.
Precise Zero Recommended
Hold the device completely still, then click. It will average the readings over about half a second for a more accurate zero.
💪
Arm Position Zeros
Set the current arm position as your starting reference. These zeros work on top of the orientation zero — move the arm to where it should read as "neutral," then click below.
Zeros wrist rotation, pointing direction, and vertical tilt all at once. This is usually all you need.
Or zero just one aspect:
Wrist Rotation
How much the wrist is rotated (like turning a doorknob). Zero this when the arm is straight and your palm faces the right direction.
Pointing Direction
Which horizontal direction the arm points (left, right, forward). Zero this when the arm is pointing in your intended forward direction.
Vertical Tilt
How far up or down the arm is angled. Zero this when the arm is at the height that should read as the neutral position.
Advanced Settings
Euler Decomposition Order Controls the mathematical order used to convert the sensor's orientation into three separate angles. Auto picks the best option to avoid "gimbal lock" problems at the time of zeroing, and is almost always the right choice.
Check Current State Ask the device to report its current zero settings — useful for troubleshooting.
